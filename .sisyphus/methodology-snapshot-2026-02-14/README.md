# Methodology Snapshot (2026-02-14)

Temporary snapshot of current thinking about Chiron's methodology layer.

This folder is intentionally *not* a source of truth. It is working documentation to help converge on a deterministic, implementable v1.

## Documents

- `execution-layers.md` - Cross-cutting execution layers and module responsibilities
- `methodology-primitives-wireframe.md` - Week 6 freeze notes for agreed methodology primitives and implementation priorities
- `bmad-full-prototype-config.md` - Full BMAD prototype config (methodology + workflow step configs + contract alignment notes)
- `bmad.md` - BMAD methodology (BMM + core) user journeys, workflow outputs, and proposed work-item mappings
- `cis.md` - CIS module journeys (design thinking, innovation strategy, problem solving, storytelling)
- `tea-testarch.md` - TEA test architecture journeys and artifacts
- `agile-scrum.md` - Proposed Agile/Scrum methodology definition for Chiron
- `ralph-loop.md` - Proposed Ralph Loop methodology definition for Chiron

## Conventions

- "From repo" means information is derived from `_bmad/**` workflow files and module catalogs.
- "Proposed" means a suggested mapping into Chiron's methodology/work-item/state-machine model.
- Artifacts are referenced by file path patterns and output folders, not embedded verbatim.
