This page indexes the work units in the default seeded methodology that are currently part of the public reference story.

## Reading order

The cleanest order is:

1. `setup`
2. `brainstorming`
3. `research`
4. `product_brief`
5. `prd`
6. `implementation`

## Work-unit map

- [Setup](/methodology/work-units/setup) — creates the initial durable baseline and launches the first downstream work.
- [Brainstorming](/methodology/work-units/brainstorming) — refines possible directions and converges on selected ones.
- [Research](/methodology/work-units/research) — gathers market, domain, or technical evidence through one branched contract.
- [Product Brief](/methodology/work-units/product-brief) — synthesizes upstream work into a concise framing document.
- [PRD](/methodology/work-units/prd) — turns framing into durable requirements and implementation draft specs.
- [Implementation](/methodology/work-units/implementation) — plans, executes, and validates code-oriented delivery.

Each reference page explains the purpose, inputs, outputs, workflow shape, steps, and lifecycle behavior of that work unit.
