This section explains Chiron's default seeded methodology.

It is the first built-in seedable method shipped in the repo. It is not the only possible future method, but it is the default public reference point today.

## What this methodology is

The current default method is a simplified augmented BMAD methodology.

It keeps the useful parts of structured planning and execution, but it expresses them through Chiron primitives instead of through a loose sequence of prompts and informal handoffs.

That means the method is modeled through:

- work units
- methodology and project facts
- work-unit facts
- workflows
- steps
- lifecycle states and transitions
- durable artifact slots

## Why this section is separate from Taskflow

The [Taskflow](/taskflow/) pages show an example project moving through runtime.

This section explains the reusable method behind that runtime path: what the seed defines, why the work units are shaped the way they are, and how the execution contracts fit together.
