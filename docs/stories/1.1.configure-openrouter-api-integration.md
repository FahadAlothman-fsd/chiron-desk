# Story 1.1: Configure OpenRouter API Integration with Provider Abstraction

## Story
**As a** developer,
**I want** to establish OpenRouter API connection with proper authentication and error handling using a provider abstraction layer,
**so that** the system can reliably access AI models through OpenRouter now and support additional providers (OpenAI, Anthropic, etc.) in the future without major refactoring.

## Acceptance Criteria
1. OpenRouter API client is implemented with proper authentication using API keys
2. API connection includes comprehensive error handling for network failures, authentication errors, and rate limiting
3. System can successfully retrieve available models list from OpenRouter API
4. API responses are properly parsed and validated against expected schema
5. Connection includes timeout configuration and retry logic for reliability
6. Error messages are user-friendly and provide actionable guidance for common issues
7. API usage is logged for debugging and monitoring purposes
8. **Provider abstraction layer exists to support future AI providers without refactoring existing code**

### Architectural Decision: Provider Abstraction
**Rationale:** This story implements a provider abstraction layer to support future AI providers (OpenAI, Anthropic, Cohere, etc.) without requiring major refactoring. While only OpenRouter is implemented now, the abstraction enables:
- Users to bring their own API keys for different providers
- DSPy to work seamlessly with multiple providers via `get_dspy_lm()` method
- Easy addition of new providers by implementing `AIProviderClient` interface
- Clean separation of concerns following Open/Closed Principle

### Provider Abstraction Interface

**AIProviderClient Abstract Base Class:**
```python
from abc import ABC, abstractmethod
from typing import AsyncGenerator, List, Dict, Optional
import dspy

class AIProviderClient(ABC):
    """
    Abstract base class for AI provider clients.
    
    This abstraction enables Chiron to support multiple AI providers
    (OpenRouter, OpenAI, Anthropic, etc.) without coupling the application
    to a specific provider implementation.
    
    Implementations must provide:
    - Model listing capabilities
    - Streaming chat completion
    - DSPy language model integration
    - Connection validation for health checks
    """
    
    @abstractmethod
    async def list_models(self) -> List[Dict[str, any]]:
        """
        Retrieve list of available models from the provider.
        
        Returns:
            List of model dictionaries with at minimum:
            - id: str - Unique model identifier
            - name: str - Human-readable model name
            
        Raises:
            AIProviderError: If model retrieval fails
        """
        pass
    
    @abstractmethod
    async def stream_chat_completion(
        self,
        messages: List[Dict[str, str]],
        model: str,
        temperature: float = 0.7,
        max_tokens: int = 2000,
        **kwargs
    ) -> AsyncGenerator[str, None]:
        """
        Stream chat completion responses from the provider.
        
        Args:
            messages: List of message dicts with 'role' and 'content'
            model: Model identifier to use
            temperature: Sampling temperature (0.0-1.0)
            max_tokens: Maximum tokens to generate
            **kwargs: Provider-specific parameters
            
        Yields:
            String chunks of the completion response
            
        Raises:
            AIProviderError: If streaming fails
            RateLimitExceededError: If rate limit is hit
        """
        pass
    
    @abstractmethod
    def get_dspy_lm(self, model: str) -> dspy.LM:
        """
        Get DSPy language model instance for this provider.
        
        This enables DSPy framework integration with different providers.
        Each provider returns the appropriate DSPy LM adapter configured
        for their API.
        
        Args:
            model: Model identifier to configure
            
        Returns:
            DSPy language model instance (e.g., dspy.OpenAI, dspy.Anthropic)
            
        Example:
            For OpenRouter: dspy.OpenAI(model=model, api_base="https://openrouter.ai/api/v1")
            For OpenAI: dspy.OpenAI(model=model)
            For Anthropic: dspy.Anthropic(model=model)
        """
        pass
    
    @abstractmethod
    async def validate_connection(self) -> bool:
        """
        Validate that the provider connection is working.
        
        Used for health checks and API key validation.
        
        Returns:
            True if connection is valid, False otherwise
        """
        pass
```

**Provider Factory Pattern:**
```python
class ProviderFactory:
    """Factory for creating AI provider client instances."""
    
    _providers = {
        "openrouter": OpenRouterClient,
        # Future: "openai": OpenAIClient,
        # Future: "anthropic": AnthropicClient,
    }
    
    @staticmethod
    def create_provider(provider_type: str, api_key: str, **config) -> AIProviderClient:
        """
        Create a provider client instance.
        
        Args:
            provider_type: Provider identifier ("openrouter", "openai", etc.)
            api_key: API key for the provider
            **config: Provider-specific configuration
            
        Returns:
            Configured AIProviderClient instance
            
        Raises:
            ValueError: If provider_type is not supported
        """
        if provider_type not in ProviderFactory._providers:
            supported = ", ".join(ProviderFactory._providers.keys())
            raise ValueError(
                f"Unsupported provider: {provider_type}. "
                f"Supported providers: {supported}"
            )
        
        provider_class = ProviderFactory._providers[provider_type]
        return provider_class(api_key=api_key, **config)
    
    @staticmethod
    def get_supported_providers() -> List[str]:
        """Get list of supported provider types."""
        return list(ProviderFactory._providers.keys())
```

### API Specifications: OpenRouter Integration
[Source: architecture/openrouter-integration.md#api-client-architecture]

**Base URL:** `https://openrouter.ai/api/v1`

**Authentication Headers:**
```python
headers = {
    "Authorization": f"Bearer {api_key}",
    "HTTP-Referer": "https://chiron.local",
    "X-Title": "Chiron",
    "Content-Type": "application/json"
}
```

**Key Endpoints:**
1. **GET /models** - Retrieve available models
   - Returns: `{"data": [{"id": "model-id", "name": "model-name", ...}]}`

2. **POST /chat/completions** - Stream chat responses
   - Request: `{"model": str, "messages": List[Dict], "temperature": float, "max_tokens": int, "stream": bool}`
   - Response: Server-Sent Events stream with format `data: {...}\ndata: [DONE]`

**Timeout Configuration:**
[Source: architecture/openrouter-integration.md#api-client-architecture]
- Default timeout: 30.0 seconds
- Connection timeout: 10.0 seconds
- Connection limits: max_connections=100, max_keepalive_connections=20

**Retry Configuration:**
[Source: architecture/openrouter-integration.md#api-client-architecture]
- Retry attempts: 3
- Wait strategy: Exponential backoff (multiplier=1, min=4s, max=10s)
- Use tenacity library: `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=4, max=10))`

**Rate Limiting:**
[Source: architecture/openrouter-integration.md#api-client-architecture]
- Default: 60 requests per minute
- Implement token bucket algorithm in `RateLimiter` class
- Check before each API request
- Raise `RateLimitExceededError` when limit exceeded

**Error Handling:**
[Source: architecture/openrouter-integration.md#api-client-architecture]
- HTTP 429 → `RateLimitExceededError("OpenRouter rate limit exceeded")`
- HTTP 401 → `ProviderAuthenticationError("Authentication failed")`
- HTTP 4xx/5xx → `OpenRouterAPIError(f"API error: {error_data}")`
- Timeout → `OpenRouterAPIError("Request timeout - OpenRouter API is slow")`
- Connection error → `OpenRouterAPIError("Connection error - Unable to reach OpenRouter")`

```python
class ChironBaseError(Exception):
    """Base error for Chiron AI service"""
    pass

class AIProviderError(ChironBaseError):
    """Base error for AI provider operations"""
    def __init__(self, message: str, provider: str = None):
        self.provider = provider
        super().__init__(message)

class OpenRouterAPIError(AIProviderError):
    """Error from OpenRouter API"""
    def __init__(self, message: str):
        super().__init__(message, provider="openrouter")

class RateLimitExceededError(AIProviderError):
    """Rate limit exceeded"""
    pass

class ProviderAuthenticationError(AIProviderError):
    """Authentication failed with provider"""
    pass
```
