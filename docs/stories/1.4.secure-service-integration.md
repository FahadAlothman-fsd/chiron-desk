# Story 1.4: Secure Service Integration

## Status
Draft

## Story
**As a** system architect,
**I want** to establish secure communication between the Hono server and AI service with request-scoped API key passing,
**so that** users can use their own API keys securely and the system can handle per-user scenarios with proper authentication and authorization.

## Acceptance Criteria
1. Secure service-to-service authentication between Hono and AI service
2. Request-scoped API key passing from Hono to AI service
3. Proper API key validation and sanitization
4. Rate limiting per user/API key combination
5. Audit logging for all API key usage
6. Error handling that doesn't expose sensitive information
7. Support for multiple providers with different authentication schemes
8. Circuit breaker pattern for resilience

## Technical Requirements

### Authentication Architecture
```typescript
// Service-to-Service Authentication
interface ServiceAuth {
  type: 'jwt' | 'mtls' | 'shared-secret';
  config: {
    issuer: string;
    audience: string;
    algorithm: string;
    keyRotationInterval: number;
  };
}

// Request Context
interface RequestContext {
  userId: string;
  apiKeyId: string;
  provider: string;
  correlationId: string;
  timestamp: number;
}
```

### API Request Flow
```
Frontend → Hono Server → Database → AI Service → OpenRouter
    ↓           ↓            ↓         ↓           ↓
  Auth     Get API Key    Pass Key   Use Key    Make Request
```

### Hono Implementation
```typescript
// Hono middleware for API key handling
app.use('/api/ai/*', async (c, next) => {
  const userId = c.get('userId');
  const apiKeyId = c.req.query('keyId') || c.req.header('X-API-Key-ID');
  
  // Get API key from database
  const apiKey = await getApiKey(userId, apiKeyId);
  
  // Forward request to AI service
  const response = await aiServiceClient.post('/chat/completions', {
    headers: {
      'Authorization': `Bearer ${apiKey.key}`,
      'X-User-ID': userId,
      'X-API-Key-ID': apiKeyId,
      'X-Correlation-ID': c.get('correlationId')
    },
    body: await c.req.json()
  });
  
  return c.json(response.data);
});
```

### AI Service Updates
```python
# FastAPI middleware for authentication
@app.middleware("http")
async def authenticate_request(request: Request, call_next):
    # Validate service token
    service_token = request.headers.get("Authorization")
    if not validate_service_token(service_token):
        raise HTTPException(401, "Invalid service token")
    
    # Extract user context
    user_id = request.headers.get("X-User-ID")
    api_key_id = request.headers.get("X-API-Key-ID")
    api_key = request.headers.get("Authorization").replace("Bearer ", "")
    
    # Add context to request state
    request.state.user_id = user_id
    request.state.api_key_id = api_key_id
    request.state.api_key = api_key
    
    response = await call_next(request)
    return response
```

## Security Requirements

### Authentication Standards
- **JWT**: RS256 signing with 15-minute expiration
- **mTLS**: Mutual TLS with certificate rotation
- **Rate Limiting**: Token bucket algorithm with Redis
- **Audit Logging**: Immutable logs with tamper detection

### Data Protection
- API keys encrypted at rest and in transit
- No API keys in logs or monitoring
- Secure key rotation without service interruption
- Data retention policies for audit logs

### Network Security
- Internal service communication over HTTPS
- Network segmentation and firewall rules
- DDoS protection and rate limiting
- Intrusion detection and prevention
