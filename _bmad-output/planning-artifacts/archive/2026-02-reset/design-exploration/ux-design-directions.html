Legend: What Each Block Means

Every direction is the same page (`Execution Orchestration Cockpit`). Only layout and interaction emphasis change. Use this legend to interpret zone labels while comparing cards.

Active Step Pane Main execution focus for the current step. Example: agent step conversation, tool calls, and step-level controls (`retry`, `resume`, `cancel`).
Diagnostics Blocked/failed explanations and remediation guidance. Example: gate mismatch (`required` vs `observed`) with evidence links.
Timeline Chronological run history. Example: transition attempts, retries, child invokes, terminal outcome.
Work Unit Queue Dependency-aware queue of pending/current work units. Example: which unit is blocked by which upstream output.
Evidence / Artifacts Persisted outputs tied to attempts. Example: generated doc snapshot, IDs, and lineage references.
Context Injection What is currently injected into step context. Example: `@work-unit`, `@artifact`, dependency refs, and mention chips in agent steps.
Command Rail / Activity Rail Power-user acceleration and event feed surfaces. Example: command palette actions, tool-call stream, and quick jump links.
Refinement Side Context Episodic methodology tuning panel. Example: adjusting transition rules while keeping execution visibility intact.
Zen Focus Behavior (Step Execution)
  • Active Step Pane expands to primary width.
  • Secondary panels (timeline, queue, artifacts, diagnostics) become collapsible drawers.
  • A compact context strip remains visible with step status, dependency count, and quick "open context" actions.
  • Context injections (`@mentions`, dependencies, artifacts) stay one-click accessible from the step pane.
